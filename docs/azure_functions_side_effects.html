F-18 Teams Notification Dispatcher Event Live Live and production verified. Single Power Automate inbound HTTP trigger per event type — 7 PA flows total, one per event. All routing logic lives in the Function — PA is a formatting and delivery layer only.

Flow: Function resolves TeamId and ChannelId from PropertyChannels table via Managed Identity (read-only DB access). POST to PA includes fully-resolved payload:
{ event, teamId, channelId, subject, body, timestamp }

PA reads event field to select the appropriate adaptive card template and POSTs to the resolved Teams channel using the provided teamId + channelId.

PA flow URLs stored as per-event Azure App Settings: PA__new_part_created, PA__new_part_in_warehouse, PA__bulk_parts_created, PA__bulk_supplies_created, PA__daily_pending_orders, PA__weekly_warehouse_report, PA__daily_channel_failures. Never hardcoded in source.

String sanitization: SanitizeForNotification helper applied at all notification body construction points (API enqueue and Function DB reads) — replaces " with in to prevent PA Adaptive Card JSON failures on user-entered strings containing inch marks or quotes.

DM channel: DatabaseManager Teams channel stored in Key Vault config (DM__TeamId + DM__ChannelId) — not in the channel registry.

Test override: NotificationOverride__TeamId + NotificationOverride__ChannelId App Settings route all dispatches to a test channel when both are set. Failure reporting suppressed when override is active.

Production verified: new_part_created, new_part_in_warehouse, bulk_parts_created, bulk_supplies_created. Daily/weekly timer flows pending natural schedule validation.
F-19 Property Channel Failure Reporter Event Live Live. DailyChannelFailuresFunction — timer-triggered daily report to the DM Teams channel summarizing any unreported PA delivery failures. Called internally by any Function that uses F-18 when PA returns a failure response.

Flow: PA failure → Function calls PATCH /property-channels/{id}/failure on the API via Managed Identity token (PropertyChannelFailure.Write App Role, configured and live). API writes LastFailedAt to PropertyChannels. Daily timer function queries unreported failures and posts consolidated report to DM channel.

Note: If NotificationOverride__TeamId + NotificationOverride__ChannelId are both active, failure reporting is skipped entirely — test channel failures must not mark production channel records. Functions remain read-only against the database — all writes route through the API.

§03

Notification Infrastructure & Data Model

Decisions made April 2026 that underpin F-18, F-19, F-20, and all future notification functions. B-11 schema is complete and deployed. B-12 (PA flow + Functions code) is next.

ℹ️ Schema migrations for PropertyChannels and ManagedProperties are deployed to production (April 22, 2026). API endpoints and PWA UI are live. All Teams channels are created. F-18 (B-12) is complete and production verified — 7 PA flows live, per-event PA__ App Settings configured in Azure. Timer-triggered flows (daily/weekly) pending natural schedule validation.
Topic Decision
PA Service Account PA flow is owned by and runs under the ATC Office licensed service account — not the Flow bot. Flow bot was rejected due to permission issues with shared channels and posting reliability.

ATC Office is a member of every team it needs to post in. This account persists regardless of personnel changes. PA Teams connector is authenticated as ATC Office — if the account password changes or MFA prompt expires, the connection must be re-authenticated in PA before flows resume.

Posts appear as "ATC Office (DO NOT REPLY)" in Teams channels.
DatabaseManager Channel Host team: IT Team
Channel type: Shared channel — allows non-IT DatabaseManager users to be invited as members without joining the IT Team itself.
Channel name: Inventory - DB Manager

ATC Office must be a member of the IT Team (host team) for posting rights. DM__TeamId in Key Vault must contain the IT Team's teamId — not the shared channel's own team reference. DM__ChannelId contains the shared channel's ID.
PA Architecture Single Power Automate inbound HTTP trigger handles all Teams notifications. Function always resolves routing before calling PA — PA receives a fully-resolved teamId and channelId in every payload. PA has zero awareness of warehouses, propCodes, or channel registry. PA reads the event field to select the adaptive card template.
Routing Protocol No incoming webhook URLs. Routing uses TeamId + ChannelId only — consistent with the existing Python oncall parser pattern used across other ATC systems. PropertyChannels table stores TeamId (nvarchar 100) and ChannelId (nvarchar 200) per warehouse channel.
PropertyChannels Table Deployed. Schema: Id, WarehouseId (FK → Warehouses), ChannelType (nvarchar 50), TeamId (nvarchar 100), ChannelId (nvarchar 200), IsActive (bit), LastFailedAt (datetime2, nullable). One warehouse → many channels.
ManagedProperties Table Deployed. Schema: Id, PropCode (unique), DisplayName, ManagedByWarehouseId (FK → Warehouses, required), Notes (nullable), CreatedDt. Satellite properties (no independent warehouse, no independent stock) are recorded here. A propCode in this table is considered reconciled by F-20 — not a gap.
PATCH /failure Auth PATCH /property-channels/{id}/failure is gated to the PropertyChannelFailure.Write App Role (not DatabaseManager). This role is assigned to the atc-inventory-functions Managed Identity via Graph API — service-to-service auth, not user-based. Configured and live.
Test Override Two Azure App Settings on the Function App (also in local.settings.json for dev):
— NotificationOverride__TeamId
— NotificationOverride__ChannelId
When both are present and non-empty, all PA dispatches use these values regardless of what PropertyChannels resolves. Failure reporting (F-19 / PATCH /property-channels/{id}/failure) is suppressed when override is active. When either is absent or empty, normal routing applies. Zero code path impact in production when unset. Values set in Azure Portal App Settings — never in source control.
PA Flow URL PA HTTP trigger URL stored as Azure App Setting: PowerAutomate__FlowUrl. Generated when the PA flow is created (B-12 Step 2). Never hardcoded in source.
Channel Routing Given a propCode: check Warehouses (self-managed) → if not found, check ManagedProperties → get ManagedByWarehouseId → query PropertyChannels where WarehouseId = [resolved] and IsActive = 1.
Failure Handling PA failure → Function calls PATCH /property-channels/{id}/failure via Managed Identity. API writes LastFailedAt. API fires DM alert. Functions remain read-only against the database — all writes route through the API per architectural rule.
Satellite Properties Properties managed by another warehouse do not receive their own warehouse record, Teams channel, or App Role. Stock is held at the managing warehouse. Financial attribution is YARDI's domain — the inventory app tracks physical location only.
YARDI Access No direct API access. YARDI pushes scheduled reports (format and delivery mechanism TBD). Virtuoso MCP connector is Claude-only beta — not suitable for production architecture. Reconciliation (F-20) is a diff-based timer function, not a real-time event consumer.
PWA Low Stock Toast Complete (B-07). supply.low_stock SignalR event includes partName and warehouseName in the payload — no cache lookup required. useLowStockToasts hook and LowStockToastStack component are live. Scoping handled by SignalR group membership.
