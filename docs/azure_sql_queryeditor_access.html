Azure SQL Query Editor Access

Procedures for granting, revoking, and auditing database access on the dev and production Azure SQL instances.

Applies To atc-inventory-db-dev · atc-inventory-db
Who Can Perform Entra Admins

01

Prerequisites

Open the Query Editor connected as yourself before performing any of the operations below.

ℹ️
Azure Portal → SQL database → Query editor → Continue as jamie@atcdevelopment.com
You must be connected as the Entra admin to create or remove Entra-based users.
02

Add a User

Full Admin (IT Staff)

sql
CREATE USER [user@atcdevelopment.com] FROM EXTERNAL PROVIDER;
ALTER ROLE db_owner ADD MEMBER [user@atcdevelopment.com];

Read-Only

sql
CREATE USER [user@atcdevelopment.com] FROM EXTERNAL PROVIDER;
ALTER ROLE db_datareader ADD MEMBER [user@atcdevelopment.com];
03

Remove a User

Drop the role membership first, then drop the user.

sql
ALTER ROLE db_owner DROP MEMBER [user@atcdevelopment.com];
DROP USER [user@atcdevelopment.com];
ℹ️
If the user was assigned db_datareader instead of db_owner, substitute accordingly in the first line.
04

Audit Current Access

sql
SELECT dp.name, dp.type_desc, r.name AS role
FROM sys.database_principals dp
JOIN sys.database_role_members rm ON dp.principal_id = rm.member_principal_id
JOIN sys.database_principals r ON rm.role_principal_id = r.principal_id
WHERE dp.type IN ('E', 'X')
ORDER BY dp.name
ℹ️
Jamie does not appear in this query. Server-level Entra admin access is not a database principal and is not visible here.

06

Notes

⚠️
Account must exist. Users must have an active @atcdevelopment.com Entra account. The FROM EXTERNAL PROVIDER clause fails immediately if the account does not exist.
ℹ️
Database-level access only. This grants database access only — no Azure Portal or resource group permissions are implied.
✅
Immediate effect. Changes take effect immediately. No token refresh required for Query Editor access. When granting access to the production database, repeat the same steps against atc-inventory-db.
