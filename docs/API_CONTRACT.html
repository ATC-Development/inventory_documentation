1. Authentication & Authorization

All protected endpoints require a valid Bearer token from Microsoft Entra ID. Include on every request:

Authorization: Bearer <access_token>
Content-Type: application/json

Five Canonical Auth Labels

LabelRequired AccessCode Pattern
AnonymousNo token requiredNo .RequireAuthorization()
AuthenticatedAny valid Entra token.RequireAuthorization()
PropertyWriteValid token + matching {propCode}_wh role or DatabaseManager.RequireAuthorization() + inline property check
DatabaseManagerValid token + DatabaseManager role.RequireAuthorization("DatabaseManager")
FunctionAppValid token + PropertyChannelFailure.Write role — Azure Functions only.RequireAuthorization("FunctionApp")

Role Tiers

TierAccessHow Assigned
DatabaseManagerFull access — all warehouses, all bulk/destructive/administrative operations. Overrides all property checks. Auto-joined to all warehouse SignalR groups.M365 group Inventory-DatabaseManagers → App Role in Azure Portal. No code change required.
{propCode}_whRead all. Write access scoped to matching warehouse only.App Role per property on ATC_InventoryAPI. See Entra_Property_Access_Management.md.
Baseline AuthenticatedRead all. Single-record creates. Scan and adjustment workflows. Part updates.Any valid Entra user with a token. No role assignment required.
FunctionAppNotification failure endpoints only.Managed Identity App Role on ATC_InventoryAPI.
Single-create vs bulk: Single creates (parts, supplies, identifiers, vendor-parts) are open to any authenticated user. Bulk and destructive operations require DatabaseManager. PUT /parts/{id} follows the single-create rule.
The PWA uses role claims for UI show/hide only. The API always enforces independently. A 403 is always possible regardless of frontend state.

3. Barcode Conventions

ATC_MASTER (Consumable Parts)

PropertyValue
Id_typeATC_MASTER
FormatZero-padded 7-digit integer string
Examples0000001, 0000042, 0001000
GenerationnextSeq.ToString().PadLeft(7, '0') — server-generated at part creation
Issue UOMAlways EA
MultiplierAlways 1
RuleCallers may NEVER supply an ATC_MASTER. Rejected with 400 ATC_MASTER_NOT_ALLOWED.

TOOL_MASTER (Equipment)

PropertyValue
Id_typeTOOL_MASTER
FormatZero-padded 7-digit integer, always starts with 8
Examples8000001, 8000042
Generation(maxSeq + 1).ToString().PadLeft(7, '0') — server-generated at equipment creation
RuleCallers may NEVER supply a TOOL_MASTER.

Unit Multiplier

unit_multiplier_to_base is a stock-in conversion factor applied at scan time only:

quantity_change = qty_packages × unit_multiplier_to_base

Never persisted to the supply quantity. Only meaningful on POST /supplies/scan/stockin.

6. SignalR Events

Hub URL: /hubs/inventory  ·  Token via query string: ?access_token=<token>

Events are cache invalidation signals only — not authoritative state. The PWA always re-fetches from the API on receipt.

The SignalR connection is additive and non-blocking. The app must function fully if the hub is unavailable. No feature may fail or hang due to a missing connection.

Warehouse Groups

Hub Method (client → server)Effect
JoinWarehouse(int warehouseId)Adds connection to warehouse_{warehouseId} group
LeaveWarehouse(int warehouseId)Removes connection from group
DatabaseManager auto-join: On OnConnectedAsync, DatabaseManager users are automatically added to every warehouse group. They do not need to call JoinWarehouse.

Events Broadcast by the API

supply.updated
→ Clients.Group("warehouse_{id}")

Any supply write: create, adjust, consume, stock-in, bulk. Payload: { warehouseId, partId }

supply.low_stock
→ Clients.Group("warehouse_{id}")

Fired when quantity is at or below min_level after any supply write. Payload: { warehouseId, partId, currentQty, minQty, partName, warehouseName }

part.created
→ Clients.All

Single part created or bulk batch completed. Payload: { partId } (single) or { count } (bulk).

part.updated
→ Clients.All

Part updated or deleted. Payload: { partId }

warehouses.changed
→ Clients.All

Warehouse created, updated, or deleted. No payload.

categories.changed
→ Clients.All

Category created, updated, or deleted. No payload.

vendors.changed
→ Clients.All

Vendor created, updated, or deleted. No payload.

equipment.changed
→ Clients.All

Any equipment lifecycle event: create, update, delete, check-out, check-in, location update. No payload.

orders.updated
→ Clients.All

Order created or status changed. No payload.

Reserved — Not Yet Broadcast

EventScopePurpose
enrichment.resolvedClients.User(oid) — targeted to triggering user via OidUserIdProviderAzure Functions enrichment completion
enrichment.failedClients.User(oid)Azure Functions enrichment failure
Broadcasts are fire-and-forget. The API emits after the DB write commits and never awaits delivery. A failed broadcast never affects the HTTP response or rolls back the write.

11. Identifiers

ATC_MASTER identifiers are server-generated. Reject any request supplying id_type: "ATC_MASTER" — error: 400 ATC_MASTER_NOT_ALLOWED.

GET /identifiers/resolve/{code} Authenticated

Resolves a scanned barcode. Returns the linked part if known. If unknown, starts an async enrichment lookup. Do not call for normal scan operations — use the dedicated supply scan endpoints. This is the entry point for the unknown-barcode flow only.

The code path parameter is normalized (lowercased and trimmed) before lookup.

// Response: 200 OK — always returns 200

// Known barcode:
{
  "status": "FOUND",
  "part": { ...PartReadDto... },
  "identifier": {
    "id": 1, "part_id": 1, "id_type": "UPC", "id_value": "012345678901",
    "part_name": "Air Filter 16x25", "issue_uom": "EA", "unit_multiplier_to_base": 1
  },
  "lookup_id": null,
  "message": null
}

// Unknown barcode (enrichment started):
{
  "status": "Lookup_Started",
  "part": null,
  "identifier": null,
  "lookup_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "message": "UPC not found. Searching web for information..."
}
// Side effects when unknown: creates IdentifierLookup row, starts async enrichment

GET /identifiers/lookup/{lookupId} Anonymous

Polls enrichment status. Intentionally unauthenticated — lookupId is a non-guessable GUID. Soft-deleted after Enrichment:RetentionDays (default 7 days).

// Response: 200 OK
{
  "lookup_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "status": "Pending",     // Pending | Complete | Exhausted | Failed | TimedOut
  "code": "012345678901",
  "result": null,          // populated when status = Complete (IdentifierEnrichResponseDto)
  "provider_attempts": null,
  "created_dt": "2026-01-01T12:00:00Z",
  "completed_dt": null
}
// Errors: 404 LOOKUP_NOT_FOUND (ID not found or past retention window)
When status = "Complete", result contains name/description suggestions and candidate identifiers. The PWA presents these to the user to confirm before calling POST /parts.

12. Supplies
