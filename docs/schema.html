Identifiers

Maps barcodes, UPCs, SKUs, and internal ATC codes to a Part. Id_value is globally unique — a barcode cannot point to two parts.

Id
int, identity
PK
Part_id
int, required
FK → Parts.Id, cascade delete. IX(Part_id)
Id_type
nvarchar, required
UPC / EAN / SKU / ATC_MASTER / TOOL_MASTER — stored uppercase
Id_value
nvarchar, required
UNIQUE globally — stored lowercase and trimmed
Issue_uom
nvarchar, required
EA / PKG / BOX / CASE / ROLL — defaults to EA
Unit_multiplier_to_base
int, required
1–100, defaults to 1. Stock-in conversion only — never persisted to supply quantity.

ATC_MASTER Convention

ℹ️
Every Part must have exactly one ATC_MASTER identifier. Auto-generated at Part creation — never manually assigned.

Format: 7-digit zero-padded integer (e.g. 0000001, 0000042). No prefix stored or scanned.
Hard rules: Unit_multiplier_to_base must always be 1. Issue_uom must always be EA.
Cap: ATC_MASTER values must remain below 8000000 to avoid collision with the TOOL_MASTER namespace.

Unit_multiplier_to_base — Stock-In Conversion Only

Applied at scan time only to compute quantity_change. Never persisted to the Supplies table.

formula
quantity_change = qty_packages × unit_multiplier_to_base

⚠️
Null uniqueness caveat: SQL Server does not treat two NULLs as equal in a unique index. Uniqueness for vendor-agnostic aliases (Vendor_id = NULL) is enforced by the API via an existence check before insert — not via a DB unique constraint. Do not add a UNIQUE constraint to (Raw_description, Vendor_id).

Normalization Rule

Raw_description is normalized before both store and lookup operations. The same logic must be applied in both paths.

csharp
private static string Normalize(string input) =>
    Regex.Replace(input.Trim().ToLowerInvariant(), @"\s+", " ");

Session Status Values

ValueMeaning
openSession active — extension is filling or has filled the cart; awaiting manager confirmation

ℹ️
Collision detection: Only one open session per vendor is permitted at a time. A second attempt from any user returns the active session details. Property managers see a read-only block; DatabaseManagers see an override option.

Session expiry: 60 minutes from Started_dt. Expired sessions do not block new sessions. A daily Azure Functions timer marks expired open sessions as abandoned.
⚠️
Callback token security: Callback_token is a GUID generated at session creation. It is transmitted to the extension via the preflight response and sent back in the extension's callback POST. It is validated server-side against the session ID, vendor ID, and method. It is never included in Application Insights logs, exception handlers, or Teams notifications.

Core Guarantees

One supply row per part+warehouse, enforced by a unique index
Barcodes/identifiers are globally unique to a single part
Every Part has exactly one ATC_MASTER identifier, auto-generated at creation
Every Equipment item has exactly one TOOL_MASTER identifier, auto-generated at creation
Name similarity checks run in the service layer before UNIQUE constraints
Transactions provide an immutable audit trail for stock changes; negative inventory is blocked in the API
Supplies always represent EA-level quantities; multiplier conversion is a stock-in operation only and is never stored
IdentifierLookup rows are never hard-deleted; soft delete + retention window governs cleanup
Orders are workflow artifacts and are never the write authority for inventory — all supply changes route through Transactions
Quote request tracking is orthogonal to the order status lifecycle — quote status does not affect order status
Only one open VendorCartSession per vendor is permitted at a time — collision detection is enforced in the API service layer, not via a DB constraint
VendorOrderMethod rows are never hard-deleted — deactivate via Is_active = 0 to preserve FK references from VendorCartSessions
