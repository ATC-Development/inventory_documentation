01

Overview

The F-10 quote request email send was originally implemented using mailto: links. Production testing confirmed mailto: is not viable in the organization's New Outlook (web wrapper) environment — sends were delayed 10+ minutes, inconsistent, and unreliable.

The replacement is Microsoft Graph API delegated send via On-Behalf-Of (OBO) flow. The user reviews a confirmation panel in the PWA, clicks Send, and the API sends the email on their behalf via Graph immediately. The email appears in the user's Sent Items and the vendor can reply directly to the user.

02

Why OBO — Not App-Only

Graph Mail.Send can be granted as either a delegated (OBO) or application (app-only) permission.

⚠️
App-only was rejected. The email From address would be the shared mailbox (inventory-quotes@atcdevelopment.com), not the user. If the vendor clicks Reply instead of Reply All, the response goes to the shared mailbox and the user never sees it directly.
✅
OBO is required so the email is sent as the logged-in user:

From: the user's M365 mailbox
CC: inventory-quotes@atcdevelopment.com (tracking mailbox) + the user themselves (immediate send confirmation)

Email appears in the user's Sent Items. Vendor reply goes directly back to the user. All ATC users have full M365 Business licenses — OBO delegated send is supported for all users.
03

Auth Flow

1 → PWA acquires API-scoped Bearer token (access_as_user → ATC_InventoryAPI)
2 → PWA calls POST /quote-requests/{token}/send with Bearer token
3 → API validates inbound token (existing Microsoft.Identity.Web pattern)
4 → API performs OBO exchange using ATC_InventoryAPI client ID + certificate from Key Vault. Scope: https://graph.microsoft.com/Mail.Send
5 → API calls Microsoft Graph sendMail on behalf of the user
6 → Email sent from user's mailbox → vendor To, tracking mailbox CC, user CC
⚠️
Critical: The PWA Bearer token is scoped to api://ATC_InventoryAPI/access_as_user. This token cannot be passed directly to Graph — its audience is the API, not Microsoft Graph. The API must perform the OBO exchange before calling Graph. Do not skip this step.

What Was Configured

1. Key Vault Certificate — ATC-InventoryAPI-GraphOBO

SettingValue

SubjectCN=ATC-InventoryAPI-GraphOBO
Validity24 months
Content TypePKCS #12

The certificate's public key (.cer) was uploaded to the ATC_InventoryAPI Entra app registration under Certificates & Secrets → Certificates.

2. Entra App Registration — ATC_InventoryAPI

Navigate to: Entra ID → App Registrations → ATC_InventoryAPI → API Permissions

PermissionTypeStatus
Mail.Send Delegated Admin consent granted ✅

3. Key Vault Secrets Added

Secret NamePurpose
GraphOBO-TenantIdEntra tenant ID — used by API for OBO token exchange
GraphOBO-ClientIdATC_InventoryAPI client ID — used by API for OBO token exchange
ℹ️
The certificate itself is retrieved from Key Vault by name (ATC-InventoryAPI-GraphOBO) at runtime — it is not stored as a secret.

4. Certificate Expiry Notification

Navigate to configure: atc-inventory-keyvault → Certificates → Certificate Contacts
Per-certificate: ATC-InventoryAPI-GraphOBO → current version → Issuance Policy → Lifetime Actions

SettingValue
Certificate ContactsIT.Department@atcdevelopment.com
Lifetime ActionEmail all contacts at 80% of lifetime (~19 months into 24 month cert)
05

Certificate Renewal Procedure

When IT receives the expiry notification email, follow these steps.

1
Generate a new certificate version in Key Vault
atc-inventory-keyvault → Certificates → ATC-InventoryAPI-GraphOBO → New Version
2
Download the new public key
Select the new version → Download in CER format.
3
Upload to Entra app registration
Entra ID → App Registrations → ATC_InventoryAPI → Certificates & Secrets → Certificates → Upload certificate
Description: Graph OBO - Mail.Send
4
Remove the old certificate
Remove the old certificate entry from the app registration after confirming the new one is active.
5
Verify
Confirm the API is still sending quote emails successfully after the next deployment or app restart. The API loads the certificate at startup via Key Vault.
⚠️
Do not skip step 4. The old certificate entry in the app registration does no harm while valid, but must be cleaned up before it expires to avoid confusion during future renewals.
06

API Configuration Required

The API (inventory_api) must be configured to retrieve the certificate from Key Vault at runtime for the OBO exchange. The following App Service configuration keys are required.

App Service KeySourceNotes
GraphOBO__TenantId Key Vault secret GraphOBO-TenantId Double-underscore for nested config
GraphOBO__ClientId Key Vault secret GraphOBO-ClientId Double-underscore for nested config
GraphOBO__CertificateName ATC-InventoryAPI-GraphOBO Name of the Key Vault certificate
GraphOBO__KeyVaultUri https://atc-inventory-keyvault.vault.azure.net/ Key Vault URI
ℹ️
The App Service Managed Identity must have Key Vault Secrets User and Key Vault Certificate User access policies on atc-inventory-keyvault to retrieve both the secrets and the certificate at runtime.
